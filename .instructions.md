## Reglas de Código

### Estándares Generales
- **Versión de C#:** C# (.NET compatible con Unity 6.3)
- **Plataforma objetivo:** Web (prioritario), mobile (future-proof)
- **Código limpio:** Nombres descriptivos, funciones de responsabilidad única
- **Documentación obligatoria:** TODOS los métodos y clases deben tener XML comments (///)
- **Null-safety:** Valida referencias antes de usarlas
- **Performance-first:** Evita GC spikes, reutiliza objetos cuando sea posible

### Archivos y Organización
- **Ubicación actual:** Todos los scripts en `Assets/Scripts/` (sin subcarpetas por ahora)
- **Estructura futura:** Se reorganizará en subcarpetas (Systems/, UI/, Battle/, Overworld/, etc.)
- **Métodos por clase:** Máximo ~15-20 para no violar SRP

## Arquitectura Actual

### Patrón Singleton
- **Managers existentes:** PartyManager, EnemyManager

- **Nota:** `DontDestroyOnLoad()` + Scene load listeners para limpiar en MainMenu

### Comunicación Entre Scripts
- **Patrón actual:** Llamadas directas (`GameObject.FindFirstObjectByType<>()`)
- **Patrón futuro:** Event-driven (events en futuras refactors)

## Convenciones de Nomenclatura

### Aplicar a código nuevo:
- Clases y métodos públicos: PascalCase
- Private methods: camelCase
- Constants: UPPER_CASE o PascalCase
- Variables privadas: camelCase
- Booleans: prefijo `is` o `has`
- Enums: PascalCase members

**Regla clave:** Nombres descriptivos (NO abreviar: `dmg` → `damage`)

## Battle System

### Estructura de Turnos
1. Create entities (party + enemies)
2. Sort by Initiative (descending)
3. For each battler: Check alive (health > 0) → Execute action
4. Remove dead battlers
5. Repeat until battle ends

### Estados
- **Start:** Inicialización
- **Selection:** Jugador elige acciones
- **Battle:** Ejecutando acciones
- **Won/Lost/Run:** Condiciones finales

### Validaciones Críticas
- SIEMPRE: if (health > 0) antes de actuar
- SIEMPRE: Remove after death
- SIEMPRE: SaveHealth() después de cambiar valores
- Usar coroutines con yield para animaciones

## Overworld y Movimiento

### Player
- Input: InputSystem (not KeyCode)
- Movimiento: Rigidbody.MovePosition()
- Animación: SetBool("isWalking")
- Dirección: Y-axis sprite rotation (0° = right, 180° = left)

### Enemy Patrol (ACTUAL)
- Waypoints tagged "Waypoint"
- Idle → Patrol → Chase (11 units) → Attack (7 units) → Die

## Performance

### NO hacer en loops:
- Crear listas nuevas
- Allocar strings sin StringToHash

### Reutilizar:
- Cacha referencias en Awake()
- Reutiliza listas (Clear en lugar de new)
- Usa StringToHash para animator parameters
